Audio-Signal Processing Tool for Parkinson's Disease Detection using Speech Recordings

Speech-based Machine Learning system that detects early-stage Parkinson's disease with 93.1% accuracy using vocal biomarkers and tree-based models.

Tech Stack: Python, Pydub, Vosk, Parselmouth, Librosa, XGBoost, Scikit-learn

Early-stage Parkinson's disease is difficult to diagnose, and existing imaging tests are expensive and often inaccurate. The goal of this project was to explore whether speech recordings alone could be used as a low-cost early detection method. The MDVR_KCL dataset provided recordings of Parkinson's patients and healthy controls, but each file included informal conversation before the scripted reading. The dataset was also small, so increasing sample size without losing signal was essential. I used the Vosk speech-to-text model to automatically detect the start of the scripted passage and remove all preceding audio. To expand the dataset, I split each 2-3 minute recording into 20-second segments, producing 145 samples. After pre-emphasis filtering, I extracted acoustic biomarkers such as jitter, shimmer, F0, MFCCs, and HNR using Parselmouth and Librosa. I trained four machine learning models (SVMs, Logistic Regression, Random Forests, and XGBoost) using GridSearchCV. Segmenting the audio significantly improved performance: full-length recordings produced 33-66% accuracy, while segmented recordings reached 91-93%. XGBoost and Random Forests achieved the highest accuracy at 93.1%. These results show that processed speech data combined with acoustic feature extraction can serve as a scalable, low-cost screening tool for early Parkinson's detection.

FinSight AI-Powered Personal Finance Assistant

AI finance assistant that turns messy transaction data into real-time insights through automated classification, natural-language querying, and parallel analytics.

Tech Stack: React, TypeScript, FastAPI, Supabase (JWT Auth, Row-Level Security), PostgreSQL, LangChain, OpenAI API

GitHub Repository

FinSight addresses a common gap in personal finance tools. People have plenty of transaction data, but it is messy, inconsistent, and difficult to interpret without manual work. I built a system that ingests this data, cleans and structures it, and produces meaningful insights in real time.

The platform uses a PostgreSQL database on Supabase, a FastAPI backend, and a React and TypeScript frontend. The backend validates uploaded transactions, classifies them with an LLM-based agent, and exposes a natural-language query agent that converts user questions into safe SQL. A separate analytics engine runs seven financial analysis tools in parallel, including spending trends, income versus expenses, category breakdowns, anomaly detection, and budget comparisons. Parallel execution reduces dashboard latency by about 70 percent and allows the system to generate a complete analytics report in a few seconds.

The frontend presents these results through a modern interface with interactive charts, budget tracking, protected routes, and a conversational AI assistant. The system provides a responsive and interpretable financial experience built on secure data handling, concurrent processing, and a full-stack architecture designed for extensibility.
